Documentação da API

Introdução

Bem-vindo à documentação da API! Aqui você encontrará informações sobre os endpoints disponíveis e como utilizá-los para interagir com o sistema.

Criar Relacionamento

O endpoint /api/create-relationship permite criar um novo relacionamento entre tabelas no banco de dados. Para utilizar este endpoint, faça uma requisição POST com os seguintes parâmetros:

  • config: Configurações do banco de dados, incluindo host, port, username, password, database e dialect.
  • fromTable: Nome da tabela de origem do relacionamento.
  • toTable: Nome da tabela de destino do relacionamento.
  • relationshipType: Tipo de relacionamento a ser criado.

Exemplo de Requisição


POST /api/create-relationship HTTP/1.1
Content-Type: application/json

{
  "config": {
    "host": "localhost",
    "port": 5432,
    "username": "admin",
    "password": "secretpassword",
    "database": "mydatabase",
    "dialect": "postgres"
  },
  "fromTable": "users",
  "toTable": "roles",
  "relationshipType": "belongsTo"
}

Parâmetros de Configuração

  • config: Configurações do banco de dados, incluindo host, port, username, password, database e dialect.

Parâmetros do Relacionamento

  • fromTable: Nome da tabela de origem do relacionamento.
  • toTable: Nome da tabela de destino do relacionamento.
  • relationshipType: Tipo de relacionamento a ser criado. Os tipos disponíveis são:
    • hasOne: Uma instância da tabela de origem possui uma única instância da tabela de destino.
    • hasMany: Uma instância da tabela de origem possui várias instâncias da tabela de destino.
    • belongsTo: Uma instância da tabela de origem pertence a uma única instância da tabela de destino.
    • belongsToMany: Uma instância da tabela de origem pode pertencer a várias instâncias da tabela de destino e vice-versa.

Resposta

  • success: Indica se o relacionamento foi criado com sucesso.
  • message: Mensagem de sucesso indicando que o relacionamento foi criado com sucesso.
