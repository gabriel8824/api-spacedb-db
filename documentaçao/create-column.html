Introdução

Bem-vindo à documentação da API! Aqui você encontrará informações sobre os endpoints disponíveis e como utilizá-los para interagir com o sistema.

Criar Coluna

O endpoint /api/create-column permite criar uma nova coluna em uma tabela existente no banco de dados. Para utilizar este endpoint, faça uma requisição POST com os seguintes parâmetros:

  • config: Objeto contendo as informações de configuração do banco de dados.
  • table: Nome da tabela onde a coluna será criada.
  • column: Objeto contendo as informações da coluna a ser criada.

Exemplo de Requisição


POST /api/create-column HTTP/1.1
Content-Type: application/json

{
  "config": {
    "host": "localhost",
    "port": 5432,
    "username": "admin",
    "password": "secretpassword",
    "database": "mydatabase",
    "dialect": "postgres"
  },
  "table": "users",
  "column": {
    "name": "age",
    "type": "integer",
    "nullable": false,
    "defaultValue": null,
    "primaryKey": false,
    "autoIncrement": false,
    "unique": false,
    "comment": "Idade do usuário",
    "validate": {
      "isInt": true,
      "min": 0
    }
  }
}

Parâmetros da Coluna

  • name: Nome da coluna.
  • type: Tipo de dados da coluna. Tipos aceitos: text, integer, decimal, double, boolean, date, datetime, json, email, password.
  • nullable: Indica se a coluna permite valores nulos (true/false).
  • defaultValue: Valor padrão para a coluna.
  • primaryKey: Indica se a coluna é uma chave primária (true/false).
  • autoIncrement: Indica se a coluna possui incremento automático (true/false).
  • unique: Indica se os valores da coluna devem ser únicos (true/false).
  • comment: Comentário sobre a coluna.
  • validate: Objeto contendo as opções de validação da coluna.

Opções de Validação

A opção validate permite especificar regras de validação para a coluna. Por exemplo, {"isInt": true, "min": 0} indica que o valor da coluna deve ser um número inteiro maior ou igual a zero. Aqui estão algumas opções de validação disponíveis:

  • isEmail: O valor deve ser um email válido.
  • isURL: O valor deve ser uma URL válida.
  • isIP: O valor deve ser um endereço IP válido.
  • isInt: O valor deve ser um número inteiro.
  • min: O valor mínimo permitido.
  • max: O valor máximo permitido.
